# 📱 移动端优先 UI 设计文档

## 🎯 设计理念

整个网站现已重新设计为**移动端优先**（Mobile-First），针对手机端访问进行了全面优化。

---

## ✨ 主要改进

### 1. 底部导航栏（移动端专属）

**位置**: 屏幕底部固定
**功能**: 
- 🎲 随机选择按钮
- ➕ 添加餐厅 / 登录按钮

**优势**:
- ✅ 拇指友好，易于单手操作
- ✅ 主要功能始终可见
- ✅ 不占用顶部空间
- ✅ 更符合移动应用习惯

```
┌─────────────────────────┐
│                         │
│   内容区域              │
│                         │
└─────────────────────────┘
┌───────────┬─────────────┐
│ 🎲 随机   │  ➕ 添加    │  ← 固定底部
└───────────┴─────────────┘
```

### 2. 更大的可触摸区域

**输入框**: 
- 移动端: `h-12` (48px) ✅
- 桌面端: `h-10` (40px)

**按钮**: 
- 移动端: `h-14` (56px) ✅
- 桌面端: `h-12` (48px)

**卡片**: 
- 更大的图片区域 (h-52 移动端)
- 更大的点击区域

### 3. 优化的筛选栏

**移动端设计**:
- 可折叠设计，节省空间
- 点击展开/收起
- 筛选条件垂直堆叠
- 大按钮，易于点击

**显示筛选数量徽章**:
```
┌─────────────────────────┐
│ 🔍 筛选  ⓵  ▼          │  ← 点击展开
└─────────────────────────┘
```

### 4. 卡片设计优化

**移动端优化**:
- ✅ 整卡可点击（跳转详情）
- ✅ 触摸反馈 (`active:scale-95`)
- ✅ 更大的图片展示
- ✅ 清晰的标题和信息层级
- ✅ 简化的操作按钮

**布局**:
```
┌─────────────────────────┐
│                         │
│   [餐厅图片 - 大]       │  ← 可点击
│                         │
├─────────────────────────┤
```

### 5. 响应式标题

**移动端**: 
- 标题: `text-xl` (20px)
- 副标题: `text-xs` (12px)

**桌面端**: 
- 标题: `text-2xl` (24px)
- 副标题: `text-sm` (14px)

### 6. 访客横幅优化

**移动端**:
- 垂直堆叠布局
- 按钮平分宽度
- 更清晰的文本层级

**桌面端**:
- 水平布局
- 紧凑的按钮组

### 7. 随机选择器

**移动端优化**:
- 全屏模态框 (95vw)
- 垂直按钮布局
- 更大的按钮 (h-14)
- 更大的emoji图标

## 📐 响应式断点

```css
sm:  640px  /* 平板起点 */
md:  768px  /* 中等屏幕 */
lg:  1024px /* 桌面起点 */
```

### 使用策略

**默认（移动端）**:
```tsx
<div className="p-4">        // 移动端 padding
<Button size="lg" className="h-14">  // 大按钮
```

**平板和桌面**:
```tsx
<div className="sm:p-6">     // 平板+ padding
<Button size="lg" className="h-14 sm:h-12">  // 响应式高度
```

## 🎨 设计原则

### 1. 拇指区域优先
- 底部导航栏（最易触达）
- 重要按钮在屏幕下半部分
- 避免顶部角落放置重要操作

### 2. 触摸友好
- 最小点击区域: 44x44px
- 按钮之间有足够间距
- 触摸反馈效果

### 3. 内容优先
- 简化的头部
- 更多垂直空间给内容
- 减少不必要的装饰

### 4. 渐进增强
- 移动端功能完整
- 桌面端增加额外功能
- 不降级，只增强

## 🔍 关键优化点

### 头部
```tsx
// 移动端: 紧凑的标题栏
<h1 className="text-xl sm:text-2xl">

// 移动端: 简化的导航
<div className="flex items-center gap-2">  // 只有语言和用户菜单
```

### 底部导航
```tsx
// 仅移动端显示
<div className="fixed bottom-0 sm:hidden">
  <Button size="lg" className="h-14">  // 大按钮
```

### 卡片交互
```tsx
// 整卡可点击 + 触摸反馈
<Card className="active:scale-95 transition-transform">
  <Link href={`/restaurants/${id}`}>
```

### 表单元素
```tsx
// 移动端更大的输入框
<Input className="h-12 text-base" />

// 移动端更大的按钮
<Button size="lg" className="h-14 text-base" />
```

## 🎯 性能优化

### 图片
- 使用 Next.js Image 组件
- 自动优化和懒加载
- 响应式图片尺寸

### 动画
- 使用 CSS transform（硬件加速）
- 简单的过渡效果
- 避免复杂动画

### 布局
- 减少重排和重绘
- 固定高度元素
- 使用 CSS Grid（性能好）

## 💡 开发建议

### 1. 移动端优先思维
```tsx
// ✅ 好的做法
<div className="p-4 sm:p-6">      // 移动端优先
<Button className="h-14 sm:h-12"> // 大到小

// ❌ 避免
<div className="p-6 max-sm:p-4">  // 桌面优先
```

### 2. 触摸目标
```tsx
// ✅ 足够大
<Button size="lg" className="h-14">

// ❌ 太小
<Button size="sm" className="h-8">
```

### 3. 视觉反馈
```tsx
// ✅ 添加触摸反馈
<Card className="active:scale-95">

// ✅ 添加加载状态
{loading && <Loader2 className="animate-spin" />}
```
